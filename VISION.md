# EVEData Project Vision

*Version 1.0 | July 2025*

## Vision

**"A New Eden where every developer can build game-changing tools and every player can access the data they need to excel."**

## Mission

To provide the fastest, most reliable, and developer-friendly data platform for EVE Online, offering unified access to real-time and historical game data through modern data products that eliminate infrastructure complexity and enable sustainable tool development.

## Reality Check

EVEData is a solo passion project built in my free time with a strict infrastructure budget under $300/month. These constraints shape every decision I make. I'm not trying to be everything to everyone – I'm focused on solving specific data access problems that have frustrated me and other EVE developers for years. Progress will be steady but measured, and I'll be transparent about what's possible within these boundaries.

## What EVEData is (and isn't)

EVEData will begin life as an API-first platform focused on solving for foundational data access and integration problems that developers and players have. It's initial focus is on creating a reliable data layer with unified access to market data, kill data, game assets, and historical information through modern, performant APIs and community access to both raw and curated datasets.

As the platform matures and proves sustainable, EVEData will thoughtfully expand into user-facing tools where it can add unique value. This might eventually include a new killboard with novel analytics, market tools with better insights, or industrial planning tools with richer connections to live data. But each expansion will be driven by genuine innovation, not just recreation. I'll build these only when EVEData can offer something meaningfully different while maintaining its core infrastructure excellence.

EVEData won't become a corporation management platform like Alliance Auth or SeAT. It won't have tools for managing members, SRP, or fleet operations. While it may eventually offer private, ISK subscription-based tools for things like multi-character management and corporation/alliance analytics, these will focus on intelligence and insights, not operational management. EVEData is about understanding and analyzing what happens in New Eden, not administering organizations within it.

## Core Values

### 1. Developer experience first

I'm building EVEData because I've felt the frustration of wrestling with the SDE and ESI and juggling multiple data sources when building EVE tools. Every decision starts with making life easier for developers. This means intuitive APIs that work on the first try, documentation that actually helps, and examples that solve real problems. Success looks like developers choosing EVEData because it genuinely saves them time – measured by adoption rate of 100+ active developers within the first year and API satisfaction scores above 4.5/5.

### 2. Trust through transparency

As a new project in an ecosystem littered with abandoned tools but also sustained by pillars like [Adam4EVE][adam4eve], [EVERef][everef], [Fuzzwork][fuzzwork], and [zKillboard][zkillboard], I know trust must be earned. I commit to open source from day one, monthly development updates (even when progress is slow), and honest communication about limitations. When I make mistakes, I'll own them publicly. Success means maintaining a public uptime dashboard showing 99.9% availability (excluding ESI downtime) and publishing monthly retrospectives that include both wins and failures.

### 3. Performance within constraints

While operating on a shoestring budget, I'm committed to delivering the best performance possible within those constraints. This means intelligent caching, efficient data engineering, and pragmatic optimization choices. I'll benchmark everything and share metrics openly. Success is sub-100ms API response times for cached data and transparent performance degradation during peak loads rather than failures. EVEData isn't trying to be the absolute fastest, but it's goal is to be consistently fast and stable enough for production use.

### 4. Sustainable by design

Too many EVE tools die from developer burnout or unsustainable economics. EVEData is designed to run indefinitely within a $300/month budget, with infrastructure costs hopefully bolstered in the future by good will donations from the community and not just my salary. Code is written for maintainability, not just functionality. Systems are documented for future contributors. Success means keeping monthly infrastructure costs under $300, achieving break-even through donations within year two, and having comprehensive documentation that enables new contributors to onboard within a day.

### 5. Community driven, thoughtfully executed

The best features will come from community pain points, but I'll balance requests with technical feasibility and project scope. I'll actively engage with developers where they are, celebrate tools built on our platform, and contribute back to the ecosystem. However, not every request will make it to the roadmap. Focus is essential for a solo project. Success looks like an active Discord with 500+ members, at least 10 production tools built on the platform, and a clear public roadmap that reflects community needs filtered through technical reality.

### 6. Progressive complexity

EVE's data is inherently complex, but accessing it doesn't have to be. EVEData will offer simple endpoints for common needs while exposing powerful capabilities for advanced users. A new developer should be able to get market data in minutes, while an alliance tool developer can access complex historical analytics. Success means having getting-started tutorials that work in under 5 minutes, tiered API complexity that serves both beginners and experts, and documentation that includes examples from "Hello, Jita" to production-scale systems.

### 7. Pragmatic innovation

Innovation for EVEData means solving real problems better, not chasing technology trends. I'll choose boring, proven technologies that will still work in five years. Success is measured by developer productivity gains, not architectural novelty. If EVEData can reduce the time to build a new market analysis tool from weeks to hours, the platform has succeeded.

## Success metrics

### Year one goals

- 100+ active developers using the platform
- 99.9% uptime (excluding ESI dependencies)
- Sub-100ms cached API response times
- Monthly infrastructure costs under $300
- Complete documentation for 100% of APIs
- 5+ production tools built on EVEData
- Referenced in EVE's official developer resources

### Long-term vision

- Become the de facto data layer for new EVE tools
- Sustainable through voluntary donations
- Active contributor community maintaining specialized data products
- EVE Online Partner status

---

*These values and boundaries guide EVEData from day one. As a solo developer with limited resources, I'm choosing focus over features, sustainability over scale, and community trust over rapid growth.*
